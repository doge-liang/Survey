# 通用负载均衡原理及其在大语言模型服务中的应用

> **快速参考**
> - 作者：陈子熙、步天赐、宋晨东、陆欣、叶寅宇、周子洁
> - 年份：2026年

> - 类别: cs.DC (分布式计算), stat.ML (机器学习)

## 摘要

本文解决了一个大型语言模型（LLM）服务系统中的关键效率问题：**超过40%的算力因Barrier同步并行处理中的负载不均衡而被浪费**。作者提出了一种名为**前瞻整数优化均衡（BF-IO）**的通用负载均衡原则，具有以下特点：

1. **将负载均衡形式化为在线整数优化问题**，具有可证明的最坏情况保证
2. **证明了负载不均衡改善随系统规模增长而增长** — 收益随集群规模扩大而增加
3. **在真实LLM服务场景中实现28%的能源减少**
4. **超越LLM服务范畴**，适用于任何具有不可迁移状态的Barrier同步并行系统

## 问题与动机

### 核心问题
在LLM推理过程中，解码阶段在G个GPU上进行数据并行运行。每个token步骤都需要同步（专家并行/张量并行中的集合通信），这意味着**整体进度由最慢的工作节点决定**。当请求响应长度发散时，各工作节点负载逐渐偏离，造成持续的落后者——较快的worker空闲耗电却无所产出。

**来自真实工业轨迹的关键数据：**
- Barrier引起的平均空闲时间：**40%**
- Barrier引起的中位数空闲时间：**41%**
- 每步浪费超过2/5的聚合算力
- 在fleet规模下，这相当于每天浪费**吉瓦时的电力**

### 三个耦合挑战
1. **未知且变化的工作负载**：分配时响应长度未知；即使有预测，准确性也很低
2. **在线、非平稳到达**：请求按顺序到达，无法预知未来的混合情况
3. **黏性分配与漂移负载**：一旦分配，请求无法迁移（KV缓存太大无法传输）

## 方法论

### BF-IO原则
在每个分配步骤，BF-IO选择**最小化近期未来负载不均衡的分配方案**，形式化为：

```
最小化  Σh=0 to H Imbalance(k+h)
约束条件：
  - 每个请求最多分配给一个worker
  - 每个worker的容量约束
  - 充分利用可用槽位
  - 二进制决策变量
```

**核心洞见**：有效的负载均衡**不需要**预测总剩余工作负载。取而代之的是依赖**短期前瞻**——预测一个进行中的任务是否会在短期内完成，比预测完整完成时间要容易得多。

### 理论保证

**定理1（均匀解码长度）**：BF-IO实现Ω(√B log G)的不均衡改善比

**定理2（非均匀/几何解码长度）**：同样的Ω(√B log G)标度关系成立

**定理3（广义非递减漂移）**：扩展到更广泛的工作负载模式，包括：
- 恒定工作负载（δk ≡ 0）
- 线性KV增长（δk ≡ 1）
- 推测解码（δk ≥ 1）
- 缓存压缩/稀疏注意力

**定理4（能源节省）**：任何不均衡改善保证都会转化为明确的能源改善。随着G → ∞，**能源节省比例超过52%**（针对现代GPU）。

### 窗口长度H
- H = 0：短视，仅当前步骤
- H 较小：平衡当前与近期未来，预测可靠
- H 较大：预测变得投机，计算成本增加
- 最优值在H ≈ 40附近（实验表明超过此值后指标趋于平稳）

## 实验结果

### 实证结果（公开 + 专有工作负载）

| 指标 | 改善 |
|------|------|
| 吞吐量 | **+90%** |
| 延迟 | **-44%** |
| 能源 | **-28%** |

### 扩展行为
- **G = 16 → 224 GPU**：
  - FCFS不均衡增长12倍（超线性）
  - BF-IO不均衡保持平稳（< 3×10⁸）
  - BF-IO在G=224时实现**14倍更低的**不均衡
  - 吞吐量：BF-IO近线性扩展（1k → 9k tokens/s），FCFS亚线性（1k → 4.5k tokens/s）
  - 能源减少：G=16时12% → G=224时**30%**

## 局限性

1. **集中式等待池抽象**：最强理论假设集中式调度器；即时分发接口可能降低性能
2. **需要非递减工作负载漂移**：不涵盖减少/高度非单调工作负载（激进剪枝、自适应压缩）
3. **单目标优化**：实际系统需要多目标处理（尾部延迟、公平性、SLO合规）

## 未来方向

- "即时分发"接口的完整理论（路由到per-worker FIFO队列）
- 扩展到减少/非单调工作负载模式
- 带有公平性和SLO约束的多目标服务

## 核心贡献

1. **通用原则**：BF-IO适用于任何具有黏性分配的Barrier同步并行系统
2. **最坏情况保证**：首次在LLM服务中建立调度策略与可证明能源效率之间的理论联系
3. **规模敏感改善**：收益随系统规模增长，恰恰在工业大规模服务系统运行的范围内
4. **实际可应用**：形式化为在线整数优化，计算上可行

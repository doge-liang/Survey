# TurboQuant：近最优失真率的在线向量量化

## 摘要

TurboQuant 是一种数据无关（data-oblivious，在线）向量量化算法，在 MSE 和内积估计两种指标下均达到**近最优失真率**，与信息论下界仅差约 2.7 倍常数因子。其核心创新是：对输入向量进行随机旋转以产生相互独立的 Beta 分布坐标，然后对每个坐标应用最优标量量化器。针对内积估计，采用两阶段方法（MSE 量化器 + 残差上的 1-bit QJL 变换）得到无偏估计器。主要应用场景包括：**KV 缓存量化**（3.5 比特/通道达到质量无差异）和**最近邻搜索**（召回率高于 PQ/RabitQ，索引时间几乎为零）。

## 问题与动机

向量量化（VQ）将高维浮点向量压缩为低比特位整数，同时保留几何结构（距离、内积）。这对于以下场景至关重要：

1. **LLM KV 缓存压缩** — 解码器式 Transformer 需要存储键值嵌入；其规模随上下文长度增长，是主要的内存瓶颈
2. **向量数据库/最近邻搜索** — 检索增强生成（RAG）和信息检索的核心技术
3. **AI 推理效率** — 加速器上 HBM 与 SRAM 之间或分布式集群间的通信瓶颈

**核心问题**：现有 VQ 方法要么（a）缺乏硬件向量化兼容、实时 AI 应用太慢，要么（b）相对于比特位的失真界不够优。此前没有任何方法在在线（数据无关）场景下、在所有比特位下达到近最优率。

## 方法论

### 核心洞察：随机旋转 + 标量量化

1. **随机旋转**输入向量，使每个坐标产生 Beta 分布（与输入数据无关）
2. 在高维空间中，不同坐标变得**近似独立**（测度集中 + 中心极限定理）
3. 这种独立性使得可以对每个坐标分别应用**最优 Lloyd-Max 标量量化器**，从而实现近最优的全局失真

### 两个变体

#### TurboQuant-MSE（算法 1）
- 优化均方误差
- 随机旋转 → Beta 分布坐标 → 每坐标 Lloyd-Max 标量量化
- 失真界：b 比特量化下 D_mse ≤ √(3π/2) · 4^(-b)

#### TurboQuant-IP（算法 2）— 内积无偏
- **关键发现**：MSE 最优量化器在内积估计中会产生偏差（b=1 时为 2/π 的乘法偏差）
- **两阶段方法**：
  1. 用 b-1 比特位应用 TurboQuant-MSE（产生小残差）
  2. 在残差上应用 1-bit 量化 Johnson-Lindenstrauss（QJL）变换
- 结果：**无偏**内积估计器，D_prod ≤ √(3π²/2) · (∥y∥²/d) · 4^(-b)

### 信息论下界（定理 3）
- 使用 Yao 极小极大原理 + 香农失真率函数
- 证明任何向量量化器必有 D_mse ≥ 4^(-b) 和 D_prod ≥ (1/d) · 4^(-b)
- TurboQuant 与这些下界仅差约 **2.7 倍**常数因子——这是一个紧致结果

## 核心创新

- **数据无关（在线）**：无需预处理或数据相关校准——适用于流式处理/KV 缓存
- **硬件友好**：简单的逐坐标标量量化 → 易于在 GPU 上向量化
- **双模式**：分离的 MSE 最优和内积无偏量化器，均有形式化保证
- **近最优**：首个达到 O(4^(-b)) 失真率且匹配信息论下界的在线量化器
- **异常值感知**：将通道分为异常值/非异常值集合，分配不同比特精度（沿用 QJL、RotateKV 等前人工作）

## 实验结果

### KV 缓存量化

| 方法 | 比特/通道 | LongBench 平均分（Llama-3.1-8B）|
|--------|-------------|-------------------------------|
| Full Cache | 16 | 50.06 |
| KIVI | 3 | 48.50 |
| PolarQuant | 3.9 | 49.78 |
| **TurboQuant** | **2.5** | **49.44** |
| **TurboQuant** | **3.5** | **50.06**（与全精度持平）|

- **3.5 比特/通道**达到质量无差异（压缩 4.5 倍）
- 2.5 比特/通道仅有轻微性能下降
- **"大海捞针"测试**：4 倍压缩的 TurboQuant 与未压缩基线召回率完全相同（0.997 分）

### 最近邻搜索（DBpedia Entities）

- **TurboQuant 4-bit**：在所有维度（d=200、1536、3072）下 recall@1@1 约 0.95
- **PQ 4-bit**：约 0.87（缺乏向量化支持）
- **RabitQ 4-bit**：约 0.88（慢，无 GPU 加速）
- **TurboQuant 索引时间**：约 0.001-0.002 秒 vs PQ（37-494 秒）和 RabitQ（597-3957 秒）——**几乎为零**

### 理论验证

- 经验 MSE 和内积误差在所有比特位（1-4）下均与理论上界吻合
- TurboQuant-IP 在所有比特位下保持无偏；TurboQuant-MSE 偏差随比特位增加而减小

## 局限性

1. **异常值处理是启发式的**：使用通道划分处理异常值（前人工作惯例），但缺乏异常值检测的形式化保证
2. **单位范数假设**：要求 ∥x∥₂=1；不满足此假设的数据集需要存储浮点范数并重新缩放
3. **非整数比特位约束**：通过通道划分实现非整数比特精度（2.5、3.5）——增加实现复杂度
4. **理论差距**：与下界仍有约 2.7 倍常数因子差距（虽小，但非零）

## 未来方向

1. 缩小与信息论下界的常数因子差距
2. 扩展到 MSE 和内积以外的其他失真度量
3. 异常值检测与量化的联合优化
4. 针对特定加速器架构的硬件专用优化

## 个人笔记

- 核心技巧：**随机旋转 → 独立坐标 → 标量量化** 这一套组合拳简洁而强大
- 两阶段（MSE 量化器 + QJL 残差）是对偏差问题的优雅解决方案
- 实际影响：3.5 比特质量无差异的 KV 缓存在 LLM 推理中有很强实用价值
- "在线/硬件友好"特性使 TurboQuant 区别于 PQ（k-means 码本）和 RabitQ（网格搜索）等前人方法
- 定理 3 的形式化下界与上界仅差约 2.7 倍——这是一个紧致的结果

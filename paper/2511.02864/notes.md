# Mathematical exploration and discovery at scale

## 摘要

AlphaEvolve 是一种 LLM 引导的进化算法，用于自动化数学发现。它将大语言模型的生成能力与自动化评估结合在迭代框架中。论文在 67 个数学问题上测试了 AlphaEvolve，涵盖分析、组合学、几何和数论等领域——在大多数情况下重现了已知最佳解决方案，并在若干案例中发现了改进的构造。值得注意的是，它能从有限情况推广到对所有输入值都成立的公式，还能与 Lean 等证明助手结合进行形式化验证。

## 问题与动机

数学发现已被计算工具彻底改变，但传统方法通常对每个问题都需要大量专家监督。作者提出 **"规模化构造数学"**——展示 AI 系统可以大规模系统地探索数学空间，而无需对每个新问题进行繁重的人工干预。

关键洞察：AlphaEvolve 能同时在多个抽象层次上运作，不仅优化参数本身，还优化发现参数的**算法策略**本身。

## 方法论

### 核心算法：LLM 引导的进化搜索

AlphaEvolve 维护一个 **Python 程序种群**，每个程序编码潜在的解决方案。每次迭代：

1. **生成器 (LLM)**：选取表现良好的程序，生成"变异"（智能的、语法感知的修改）
2. **评估器**：运行程序，根据目标分配数值分数
3. **选择**：最佳程序存活到下一代

### 两种运行模式

| 模式 | 目的 | 使用场景 |
|------|------|----------|
| **搜索模式** | 为固定问题规模找到最佳构造 | 优化数值常数 |
| **推广模式** | 找到对所有输入规模都成立的公式 | 发现可推广的模式 |

### 与 FunSearch 的对比

| 能力 | FunSearch | AlphaEvolve |
|------|-----------|------------|
| 进化对象 | 单个函数 | 整个代码文件 |
| 代码长度 | 10-20 行 | 数百行 |
| 语言 | Python | 任意语言 |
| 评估时间 | ≤20 分钟 (单 CPU) | 数小时，并行，加速器 |
| LLM 样本数 | 数百万 | 数千个足够 |
| 模型规模 | 小模型 | 从 SotA LLM 受益 |
| 上下文 | 最小 | 丰富的上下文和反馈 |
| 指标 | 单个 | 多指标同时优化 |

### 与证明助手的流水线整合

作者展示了强大的工作流：
```
AlphaEvolve（模式发现）→ Deep Think（符号证明）→ AlphaProof（Lean 形式化验证）
```

这条流水线在有限域 Kakeya 问题上完成了从模式观察到形式化验证定理的完整闭环。

## 主要成果

### AlphaEvolve 匹配或超越先前工作的案例：

- **有限域 Kakeya 问题**：发现的构造启发了新证明
- **Nikodym 问题**：构造启发了 Tao 的新论文
- **算术 Kakeya 问题**：结果启发了另一篇论文
- **网格上的等腰三角形**：找到 112 点配置（先前最佳是 110，经过数月尝试）
- **球面调和积分**：发现精确表达式
- **环负载问题**：将下界从 ~1.1 改进到 1.119
- **3D 沙发问题**：找到体积 ≥1.81（严格证明)，估计 1.84（超越先前候选）
- **IMO 2025 第 6 题**：为所有完美平方网格找到最优铺砌构造

### 规模化特性

- 每个问题的平均设置时间：**仅数小时**
- 传统方法通常需要长得多的时间
- 系统可扩展到同时研究大型问题类

## 局限性

1. **仅适用于可优化问题**：在目标为可爬山的平滑分数函数时效果最佳
2. **可解释性权衡**：进化的搜索启发式可能不透明，即使最终构造可解释
3. **非通用**：无法证明定理或解决没有清晰优化目标的问题
4. **仍需专家设置**：问题表述仍需专家参与

## 关键创新

1. **元层次进化**：优化优化过程本身
2. **搜索模式**：进化搜索构造的程序，而非直接进化构造
3. **多指标优化**：同时优化多个目标
4. **AI 流水线整合**：将 AlphaEvolve 与形式化验证系统结合

## 个人思考

这篇论文代表了 **AI 辅助数学研究** 的重要一步。关键洞察是 LLM 可以引导进化搜索，不仅找到解决方案，而且找到人类可以理解、推广和证明的**可理解的**解决方案。

AlphaEvolve（发现）、Deep Think（证明）和 AlphaProof（验证）之间的协作表明，AI 系统可以将数学猜想从模式观察到形式化验证定理的过程规模化，最小化人工干预。

"规模化构造数学" 的表述很贴切——这不是 AI "解决" 数学，而是 AI 作为增强人类数学直觉的强大工具。
